Knowledge API — машиночитаемый граф знаний

Вся база знаний платформы о казначейском сопровождении в JSON: 487 сущностей (термины, вопросы, документы, ошибки, мастера, центры, услуги, законы) и 4187 типизированных связей. Данные генерируются автоматически из тех же источников, что и страницы сайта, — расхождения исключены конструкцией. Использование: AI-агенты, интеграции, исследования; при цитировании ссылайтесь на источник.

Эндпоинты

  • /api/knowledge-graph.json — полный граф (узлы + рёбра); .min
  • /api/entities.json — все сущности; по слоям: glossary · faq · documents · errors · tools · hubs · services
  • /api/entity-index.json — поиск: имя/алиас/аббревиатура → id · entity-map.json
  • /api/relations.json — рёбра · graph-statistics.json · registry.json
  • /entity/<id>.json — карточка сущности (пример: /entity/ls-71.json); /entity/index.json
  • Текстовые AI-выгрузки: llms.txt · llms-full.txt · llms-entities.txt · llms-{glossary,faq,documents,errors,tools,services}.txt

Формат сущности

id · type[] · name · aliases[] · summary (AI Summary) · citation (краткое определение) · pages[] (носители) · relations{RELATED_TERM, REQUIRES_DOC, SEE_ERROR, SEE_FAQ, PART_OF_TOPIC, MONETIZED_BY, DIAGNOSED_BY, REGULATED_BY, …} · scores{authority, coverage, connectivity, citation, aiReadiness, density} · schema_id (@id JSON-LD сети сайта) · updated. Схема: описана в GRAPH_SCHEMA.
